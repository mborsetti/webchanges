# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please alert us by opening a [discussion](https://github.com/mborsetti/webchanges/discussions).

Please include *only* the following details in your report:

*   Type of issue (e.g., buffer overflow, SQL injection, etc.)
*   An email address where you can be reached

If required, we will contact you via email for the following information:

*   Full paths of source file(s) related to the manifestation of the issue
*   The location of the affected source code (tag/branch/commit or direct URL)
*   Any special configuration required to reproduce the issue
*   Step-by-step instructions to reproduce the issue
*   Proof-of-concept or exploit code (if possible)
